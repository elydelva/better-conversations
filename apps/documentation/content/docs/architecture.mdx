## Package Layers

```
types/         → Shared domain types (Chatter, Conversation, Block, PolicyObject, etc.). Used by core, client, adapters.
core/          → Pure engine. Interfaces. Hook pipeline. Permission logic. Zero external imports.
adapters/      → Concrete implementations of core interfaces. Drizzle, Prisma, MongoDB.
handlers/      → Optional HTTP bridges. Next.js, Hono, Express. Take a ConversationEngine, expose routes.
blocks/        → Block plugins (media, reaction, embed, poll). Each enriches the core type registry.
roles/         → Role plugins (moderator, admin). Each carries a policy and registers in DB.
client/        → Type-safe browser SDK. createBetterConversationsClient with plugins (sseClient, historyClient).
plugins/       → Extensions: SSE, presence, history, audit log, rate limiting. Plugins can export client-side plugins too.
cli/           → Schema generation and migrations. bc generate, bc migrate.
```

## Types Package

`@better-conversation/types` provides shared domain types used across the ecosystem:

- `Chatter`, `ChatterInput`
- `Conversation`, `ConversationInput`, `ConversationFilters`
- `Participant`, `ParticipantInput`
- `Block`, `BlockInput`, `BlockFilters`, `BlockStatus`, `ConversationStatus`
- `Paginated`, `PolicyObject`, `ResolvedPolicy`

Import from `@better-conversation/types` or `@better-conversation/core` (re-exports). Plugin-specific types (e.g. `PresenceInfo`, `BlockHistoryEntry`) are exported from their respective packages.

## Extensibility

The engine is designed for plugins that add:

- **Routes** — Merged with core routes (e.g. `/conversations/:id/stream` from plugin-sse)
- **Hooks** — Merged with config hooks (e.g. `onBlockAfterUpdate` from plugin-history)
- **Services** — Attached to engine via `createServices` (e.g. `engine.getPlugin("presence")`)
- **Schema** — Tables and column extensions via `schemaContribution` (Schema Language)

## DatabaseAdapter Interface

The core defines this interface. The adapter implements it. The core only knows the interface.

| Area | Methods |
|------|---------|
| **chatters** | `find`, `findByEntity`, `create`, `update` |
| **conversations** | `find`, `findByEntity`, `list`, `create`, `update` |
| **participants** | `list`, `find`, `add`, `update`, `remove` |
| **blocks** | `find`, `list`, `create`, `update`, `softDelete` |
| **permissions** | `check`, `grant`, `revoke` |

## ConversationEngine

The engine operates solely via `config.adapter`. It does not know if you're using Drizzle, Prisma, or an in-memory mock.

```ts title="ConversationEngine"
export class ConversationEngine<
  TBlocks extends BlockRegistry = DefaultBlockRegistry,
  TRoles extends RoleRegistry = DefaultRoleRegistry,
> {
  constructor(private readonly config: ConversationConfig<TBlocks, TRoles>) {}
}
```

## Config — Single Entry Point

```ts title="ConversationConfig"
export interface ConversationConfig<TBlocks, TRoles> {
  adapter: DatabaseAdapter;
  additionalBlocks?: TBlocks;
  additionalRoles?: TRoles;
  hooks?: ConversationHooks<TBlocks, TRoles>;
  policies?: PolicyConfig<TRoles>;
  plugins?: ConversationPlugin[];
  tablePrefix?: string;  // default: "bc_"
  generateId?: () => string;
}
```

## Design Principles

1. **Core pure** — No framework, ORM, or runtime in core. Testable without DB or HTTP.
2. **Interfaces-first** — Core defines interfaces; adapters implement them.
3. **Blocks = registry** — Core only knows `text`. Each additional block enriches the registry.
4. **Hooks = pipeline** — Every action passes through outcomes: `next`, `refuse`, `transform`, `flag`, `defer`, `queue`.
5. **Zero vendor** — No external service required. Realtime is opt-in.
6. **Type-safety end-to-end** — Block metadata types flow from `createBlock()` to the browser client.
7. **Policies = merge tree** — Rules compose by level. More specific scope wins. Defaults are safe.
8. **Schema language** — Core defines an ORM-agnostic schema format; adapters translate to Drizzle, Prisma, etc. Plugins extend the schema without knowing the ORM.
