The Thread plugin provides thread-specific policies and hooks, allowing you to control threading behavior across your conversations.

## Installation

```bash
bun add @better-conversation/plugin-thread
```

## Setup

```ts title="engine.ts"
import { betterConversation } from "@better-conversation/core";
import { createThreadPlugin } from "@better-conversation/plugin-thread";

const engine = betterConversation({
  adapter,
  plugins: [
    createThreadPlugin()
  ],
});
```

## Structure

```
Root block (threadParentId: null)
├── Reply 1 (threadParentId: root.id)
│   ├── Reply 1.1
│   └── Reply 1.2
└── Reply 2 (threadParentId: root.id)
```

## Send a Reply

```ts title="example.ts"
const reply = await client.blocks.send(conversationId, {
  authorId: chatterId,
  type: "text",
  body: "This is a reply",
  threadParentId: parentBlockId,
});
```

## List Thread Replies

```ts title="example.ts"
const { items } = await client.blocks.list(conversationId, {
  threadParentId: parentBlockId,
  limit: 20,
});
```

## Policy Levels

Threads have their own policy level. When resolving policy for a block in a thread:

1. Global
2. Role (participant's role)
3. Chatter
4. Conversation
5. **Thread** (scope: `threadParentBlockId`)

Thread-specific policies allow different rules per thread (e.g. locked thread, rate limits).

## Thread Limits

Policy fields that affect threads:

| Field | Description | Default |
|-------|-------------|---------|
| `threadsEnabled` | Whether threads are allowed | `true` |
| `maxThreadDepth` | Max nesting depth (1 = one level of replies) | `1` |
| `maxThreadReplies` | Max replies per parent | — |
| `threadClosed` | Thread closed (no new replies) | `false` |

## API

The Thread plugin adds a route to set policies specifically for a thread (a parent block and its replies).

| Method | Path | Description |
|--------|------|-------------|
| `PATCH` | `/policies/conversations/:id/threads/:blockId` | Set a policy for a thread |

## Hooks

- `onThreadCreated(ctx, outcomes)`: Fired when the first reply in a thread is sent. You can block thread creation by returning `outcomes.refuse()`.
