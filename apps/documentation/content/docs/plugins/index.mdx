title: Plugins
description: Extend the engine with blocks, roles, hooks, and custom behavior via ConversationPlugin.
---

```ts title="ConversationPlugin"
interface ConversationPlugin {
  name: string;
  version?: string;
  createServices?(engine, config): Record<string, unknown>;
  routes?: Route[];
  hooks?: Partial<ConversationHooks>;
  schemaContribution?: SchemaContribution;
  init?(engine: unknown): Promise<void> | void;
}
```

## Adding a Plugin

```ts title="lib/conversation.ts"
import { betterConversation } from "@better-conversation/core";
import { createSsePlugin } from "@better-conversation/plugin-sse";

const conv = betterConversation({
  adapter,
  plugins: [createSsePlugin()],
});
```

## Plugin Packages

| Plugin | Package | Description |
|--------|---------|--------------|
| [SSE](/docs/plugins/sse) | `@better-conversation/plugin-sse` | Real-time block updates via Server-Sent Events |
| [Presence](/docs/plugins/presence) | `@better-conversation/plugin-presence` | Mark-read, typing status, last-seen |
| [History](/docs/plugins/history) | `@better-conversation/plugin-history` | Block edit history |
| [Rate Limit](/docs/plugins/rate-limit) | `@better-conversation/plugin-rate-limit` | Blocks per chatter per conversation |
| [Audit](/docs/audit) | `@better-conversation/core` (built-in) | Immutable audit log via `config.audit` |
| [Blocks](/docs/blocks) | `@better-conversation/blocks` | Media, reaction, embed, poll block types |
| [Roles](/docs/roles) | `@better-conversation/roles` | Moderator, admin, guest, support roles |

## Custom Plugins

Create a plugin that implements `init` and optionally returns hooks to merge into config:

```ts title="lib/plugins/my-plugin.ts"
import type { ConversationPlugin, ConversationHooks } from "@better-conversation/core";

export const myPlugin: ConversationPlugin = {
  name: "my-plugin",
  async init(engine) {
    // Register custom logic, extend engine
  },
};

// Or a hook-based plugin
export function createMyPlugin(): ConversationPlugin & { hooks: ConversationHooks } {
  return {
    name: "my-plugin",
    hooks: {
      onBlockBeforeSend: async (ctx, outcomes) => {
        // Custom logic
        return outcomes.next();
      },
    },
  };
}
```

For hook-based plugins, merge the returned hooks into your config:

```ts title="lib/conversation.ts"
const myPlugin = createMyPlugin();
betterConversation({
  adapter,
  hooks: {
    ...myPlugin.hooks,
    onBlockBeforeSend: composeHooks(myPlugin.hooks.onBlockBeforeSend, myHook),
  },
});
```
