## Features

| Feature | Description |
|---------|-------------|
| **Chatters** | Identity abstraction — users, bots, organizations |
| **Conversations** | Grouped threads with participants and metadata |
| **Blocks** | Typed content units (text, media, reactions, custom) |
| **Hooks** | Moderation, rate limiting, enrichment, side effects |
| **Policies** | Composable authorization rules with merge tree (global → role → chatter → conversation → thread) |
| **SDK** | Server and client APIs, fully typed end-to-end |

## Core Principles

- **Pure core** — The core has no knowledge of Express, Next.js, Drizzle, or PostgreSQL. It never imports them. It works exclusively with **TypeScript interfaces** that you inject via config.
- **Blocks by default** — Text only out of the box; additional blocks come from opt-in plugins.
- **Hooks** — Pipeline with outcomes: `next`, `refuse`, `transform`, `flag`, `defer`, `queue`.
- **Policies** — Multi-level merge from global → role → chatter → conversation → thread.
- **Adapters** — Drizzle, Prisma, Mongo; you choose the storage backend.

## Runtime Support

Works on Node, Bun, Deno, Edge, and workers. Zero external service required; realtime (SSE) is opt-in.
